# Manuscript Structure Reference

## Directory Layout

```
BookProject/
├── Introduction.md              # Opening chapter
├── Part1_Foundation/            # Part directories
│   ├── Chapter01_Topic.md
│   ├── Chapter02_Topic.md
│   └── Chapter03_Topic.md
├── Part2_System/
│   ├── Chapter04_Topic.md
│   ├── Chapter05_Topic.md
│   └── ...
├── Part3_Growth/
│   └── ...
├── Part4_Protection/
│   └── ...
├── Part5_Execution/
│   └── ...
├── BackMatter.md                # Resources, About Author, Index
└── publishing/
    ├── compile_book.py          # Compilation script
    ├── generate_pdf.py          # PDF generator
    ├── generate_index.py        # Index generator
    ├── front_matter.md          # Optional separate front matter
    ├── cover_front.html         # Optional HTML cover design
    ├── cover_back.html          # Optional HTML back cover
    ├── manuscript_compiled.md   # Generated output
    └── BookTitle_v*.pdf         # Generated PDFs
```

## File Naming Conventions

### Chapter Files
```
Chapter##_ShortTitle.md
```
- Two-digit chapter number (01, 02, etc.)
- Underscore separator
- CamelCase or snake_case title
- Examples:
  - `Chapter01_Money_Beliefs.md`
  - `Chapter07_Automation.md`
  - `Chapter15_90Day_Plan.md`

### Part Directories
```
Part#_PartName/
```
- Part number (1-5)
- Underscore separator
- Descriptive name
- Examples:
  - `Part1_Foundation/`
  - `Part2_System/`
  - `Part3_Growth/`

## Chapter Structure

### Standard Chapter Format

```markdown
# Chapter 1: Chapter Title

Opening hook paragraph that draws the reader in...

## First Major Section

Content for this section...

### Subsection Title

More detailed content...

## Second Major Section

Continue with main content...

### Another Subsection

Details...

## Key Takeaways

**Key Takeaway:** Summary of the main point.

## Exercises

- [ ] First exercise item
- [ ] Second exercise item
- [x] Completed example

## Chapter Summary

Concluding thoughts that tie back to the opening...
```

### Chapter Header Levels

| Level | Markdown | Use |
|-------|----------|-----|
| H1 | `# ` | Chapter title only |
| H2 | `## ` | Major sections |
| H3 | `### ` | Subsections |
| H4 | `#### ` | Sub-subsections (use sparingly) |

## Part Divider Structure

Parts are inserted by the compile script, not as separate files:

```python
# In compile_book.py FILE_ORDER:
("Part1_Foundation/Chapter01_Topic.md",
 "\n\\newpage\n\n# PART I: THE FOUNDATION\n\n*Building the Mental Architecture for Wealth*\n\n\\newpage\n\n"),
```

This creates:
```markdown
# PART I: THE FOUNDATION

*Building the Mental Architecture for Wealth*
```

## Compile Script Configuration

### FILE_ORDER Array

```python
FILE_ORDER = [
    # (filename, prefix_text)

    # Introduction (no part prefix)
    ("Introduction.md", ""),

    # Part 1 - first chapter gets part header
    ("Part1_Foundation/Chapter01_Topic.md",
     "\n\\newpage\n\n# PART I: THE FOUNDATION\n\n"
     "*Part subtitle here*\n\n\\newpage\n\n"),

    # Subsequent chapters in part - just page break
    ("Part1_Foundation/Chapter02_Topic.md", "\n\\newpage\n\n"),
    ("Part1_Foundation/Chapter03_Topic.md", "\n\\newpage\n\n"),

    # Part 2
    ("Part2_System/Chapter04_Topic.md",
     "\n\\newpage\n\n# PART II: THE SYSTEM\n\n"
     "*Part subtitle here*\n\n\\newpage\n\n"),
    # ... more chapters

    # Back Matter
    ("BackMatter.md", "\n\\newpage\n\n"),
]
```

## Special Considerations

### Unicode Characters
Avoid or ensure sanitization of:
- Smart quotes (" " ' ')
- Em/en dashes (— –)
- Ellipsis (…)
- Bullets (•)
- Special symbols

The PDF generator converts these to ASCII equivalents.

### Page Breaks
Use `\newpage` for explicit breaks:
```markdown
End of chapter content.

\newpage

Start of next chapter.
```

### Cross-References
Use consistent naming:
```markdown
As discussed in Chapter 3...
See the Resources section for...
Refer to the Account Architecture chapter...
```

### Images
If including images:
```markdown
![Alt text](images/filename.png)
```
Store in `images/` directory at project root.
